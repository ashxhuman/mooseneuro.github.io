PIDController
=============


PID feedback controller.PID stands for Proportional-Integral-Derivative. It is used to feedback control dynamical systems. It tries to create a feedback output such that the sensed (measured) parameter is held at command value. Refer to wikipedia (http://wikipedia.org) for details on PID Controller.

.. describe:: gain (type: double, class: PIDController)

   This is the proportional gain (Kp). This tuning parameter scales the
   proportional term. Larger gain usually results in faster response, but
   too much will lead to instability and oscillation.

   index (dataId) of self.

.. describe:: integral (type: double, class: PIDController)

   The integral term. It is calculated as INTEGRAL(error dt) = previous_integral
   + dt * (error + e_previous)/2.

   For a FieldElement: number of entries of self.For a regular Element:
   One.

.. describe:: outputValue (type: double, class: PIDController)

   Output of the PIDController. This is given by: gain * ( error + INTEGRAL[
   error dt ] / tau_i + tau_d * d(error)/dt ) Where gain = proportional
   gain (Kp), tau_i = integral gain (Kp/Ki) and tau_d = derivative gain
   (Kd/Kp). In control theory this is also known as the manipulated variable
   (MV)

.. describe:: parent (type: ObjId, class: PIDController)

   Parent ObjId for current object

.. describe:: path (type: string, class: PIDController)

   text path for object

.. describe:: saturation (type: double, class: PIDController)

   Bound on the permissible range of output. Defaults to maximum double
   value.

.. describe:: sensed (type: double, class: PIDController)

   Sensed (measured) value. This is commonly known as process variable(PV)
   in control theory.

.. describe:: sourceFields (type: vector<string>, class: PIDController)

   List of all source fields on Element, that is fields that can act as
   message sources.

.. describe:: tauD (type: double, class: PIDController)

   The differentiation time constant, typically = dt / 4. This is derivative
   gain (Kd) times proportional gain (Kp). Larger Kd (tauD) decreases overshoot
   at the cost of slowing down transient response and may lead to instability.

.. describe:: tauI (type: double, class: PIDController)

   The integration time constant, typically = dt. This is actually proportional
   gain divided by integral gain (Kp/Ki)). Larger Ki (smaller tauI) usually
   leads to fast elimination of steady state errors at the cost of larger
   overshoot.

.. describe:: this (type: 7Neutral, class: PIDController)

   Access function for entire object

.. describe:: tick (type: int, class: PIDController)

   Clock tick for this Element for periodic execution in the main simulation
   event loop. A default is normally assigned, based on object class, but
   one can override to any value between 0 and 19. Assigning to -1 means
   that the object is disabled and will not be called during simulation
   execution The actual timestep (dt) belonging to a clock tick is defined
   by the Clock object.

.. describe:: valueFields (type: vector<string>, class: PIDController)

   List of all value fields on Element.These fields are accessed through
   the assignment operations in the Python interface.These fields may also
   be accessed as functions through the set<FieldName> and get<FieldName>
   commands.

.. describe:: proc (type: void, class: PIDController)

   This is a shared message to receive Process messages from the scheduler
   objects.The first entry in the shared msg is a MsgDest for the Process
   operation. It has a single argument, ProcInfo, which holds lots of information
   about current time, thread, dt and so on. The second entry is a MsgDest
   for the Reinit operation. It also uses ProcInfo.
