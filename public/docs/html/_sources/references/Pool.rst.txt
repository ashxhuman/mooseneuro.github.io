Pool
====


Pool of molecules of a given species.

Author:   Upi Bhalla


Attributes:
-----------



Value Attributes:
-----------------

.. describe:: children (type: vector<Id>, class: Pool)

   vector of ObjIds listing all children of current object

.. describe:: className (type: string, class: Pool)

   Class Name of object

.. describe:: compartment (type: ObjId, class: Pool)

   ObjId of parent compartment of pool. If the compartment isn'tavailable
   this returns the root ObjId.

.. describe:: conc (type: double, class: Pool)

   Concentration of molecules in this pool

.. describe:: concInit (type: double, class: Pool)

   Initial value of molecular concentration in pool

.. describe:: coords (type: vector<double>, class: Pool)

   coordinates of pool, based on mid-point of voxel in which it is situated.
   If the parent compartment isn'tavailable this returns [0, 0, 0].

.. describe:: destFields (type: vector<string>, class: Pool)

   List of all destination fields on Element, that is, fieldsthat are accessed
   as Element functions.

.. describe:: diffConst (type: double, class: Pool)

   Diffusion constant of molecule

.. describe:: dt (type: double, class: Pool)

   Timestep used for this Element. Zero if not scheduled.

.. describe:: fieldIndex (type: unsigned int, class: Pool)

   For a FieldElement: field Index of self.For a regular Element: zero.

.. describe:: idValue (type: unsigned int, class: Pool)

   Object id of self, converted to an unsigned int.

.. describe:: index (type: unsigned int, class: Pool)

   For a FieldElement: Object index of parent.For a regular Element: Object
   index (dataId) of self.

.. describe:: isBuffered (type: bool, class: Pool)

   Flag: True if Pool is buffered. In the case of Pool and BufPool the field
   can be assigned, to change the type of the Pool object to BufPool, or
   vice versa. None of the messages are affected. This object class flip
   can only be done in the non-zombified form of the Pool/BufPool. In Zombies
   it is read-only.

.. describe:: me (type: ObjId, class: Pool)

   ObjId for current object

.. describe:: midPoint (type: vector<double>, class: Pool)

   mid-point of voxel in which Pool is situated. If the parent compartment
   isn'tavailable this returns [0, 0, 0].

.. describe:: motorConst (type: double, class: Pool)

   Motor transport rate molecule. + is away from soma, - is towards soma.
   Only relevant for ZombiePool subclasses.

.. describe:: msgIn (type: vector<ObjId>, class: Pool)

   Messages coming in to this Element

.. describe:: msgOut (type: vector<ObjId>, class: Pool)

   Messages going out from this Element

.. describe:: n (type: double, class: Pool)

   Number of molecules in pool

.. describe:: nInit (type: double, class: Pool)

   Initial value of number of molecules in pool

.. describe:: name (type: string, class: Pool)

   Name of object

.. describe:: numData (type: unsigned int, class: Pool)

   # of Data entries on Element.Note that on a FieldElement this does NOT
   refer to field entries,but to the number of DataEntries on the parent
   of the FieldElement.

.. describe:: numField (type: unsigned int, class: Pool)

   For a FieldElement: number of entries of self.For a regular Element:
   One.

.. describe:: parent (type: ObjId, class: Pool)

   Parent ObjId for current object

.. describe:: path (type: string, class: Pool)

   text path for object

.. describe:: sourceFields (type: vector<string>, class: Pool)

   List of all source fields on Element, that is fields that can act as
   message sources.

.. describe:: speciesId (type: unsigned int, class: Pool)

   Species identifier for this mol pool. Eventually link to ontology.

.. describe:: this (type: 7Neutral, class: Pool)

   Access function for entire object

.. describe:: tick (type: int, class: Pool)

   Clock tick for this Element for periodic execution in the main simulation
   event loop. A default is normally assigned, based on object class, but
   one can override to any value between 0 and 19. Assigning to -1 means
   that the object is disabled and will not be called during simulation
   execution The actual timestep (dt) belonging to a clock tick is defined
   by the Clock object.

.. describe:: valueFields (type: vector<string>, class: Pool)

   List of all value fields on Element.These fields are accessed through
   the assignment operations in the Python interface.These fields may also
   be accessed as functions through the set<FieldName> and get<FieldName>
   commands.

.. describe:: volume (type: double, class: Pool)

   Volume of compartment. Units are SI. Utility field, the actual volume
   info is stored on a volume mesh entry in the parent compartment.This
   mapping is implicit: the parent compartment must be somewhere up the
   element tree, and must have matching mesh entries. If the compartment
   isn'tavailable the volume is just taken as 1
