HHGateF2D
=========


HHGateF2D: Gate for Hodkgin-Huxley type channels, equivalent to the m and h terms on the Na squid channel and the n term on K.
This is specialized for dependency on two variables, voltage and concentration (usually [Ca2+]).
Unlike HHGate2D, HHGateF2D evaluates the formulas for the gate parameters directly. This is slower than HHGate2D's interpolation-table lookup, but numerically more accurate, which can be important when the concentration can vary in an exponential scale. It also saves one from the large memory requirement for storing large 2D arrays.
The formulas must be specified as in the form f(v, c), where v and c are the variable names. For example:
"1500/(1 + (c / 1.5e-4 * exp(-77 * v)))"
While the names correspond to voltage and concentration, they can be any two parameters mapped to the corresponding fields in HHChannelF2D.
Additionally, like HHGateF, HHGateF2D provides the following  predefined variable names to facilitate intermediate  calculations:
`alpha` for forward rate,
`beta` for backward rate,
`tau` for time constant, and
`inf` for steady state open fraction
as per Hodgkin and Huxley's formulation.

by Niraj Dudani, 2009, NCBS.

.. describe:: alphaExpr (type: string, class: HHGateF2D)

   Expression for voltage-dependent rates, forward rate `alpha`. This requires
   the expression for `beta` to be defined as well. The syntax follows exprtk,
   with variable name `v` for input variable (which can be voltage or concentration
   depending on message connection in case of HHGateF which takes only one
   input). For HHGateF2D which depends on two inputs, the variable names
   are `v` for voltage, and `c` for concentration. And additional set of
   variable names are available for cases that require intermediate calculations.
   These are: `alpha` for forward rate, `beta` for backward rate, `tau`
   for time constant, and `inf` for steady state open fraction as per Hodgkin
   and Huxley's formulation. This is useful for conditional values for these
   parameters: Example: ~(alpha:=0.3 * exp(-80 * (v -(-46e-3))) + 3.5, alpha
   < 3.8? 3.8: alpha) first computes `alpha` by the first formula, and returns
   it only if the computed value is >= 3.8, otherwise it returns 3.8.

.. describe:: betaExpr (type: string, class: HHGateF2D)

   Expression for voltage-dependent rates, backward rate `beta`. This requires
   the expression for `alpha` to be defined as well. See documentation on
   `alpha` for details on predefined variable names.

.. describe:: tauExpr (type: string, class: HHGateF2D)

   Expression for voltage-dependent rates, time constant `tau`. This requires
   the expression for `inf` to be defined as well. See documentation for
   `alpha` for details on predefined variable names. Example of a complex
   conditional expression (based on Maex and De Schutter 1998): ~(alpha
   := 750 * exp(81 * (v - (-39e-3))), beta := 750 * exp(-66 * (v - (-39e-3))),
   tau := 1/(alpha + beta), tau < 1e-5? 1e-5) This computes alpha and beta
   and then from those, tau. However if the calculated value of tau falls
   under 1e-5, it makes the value 1e-5.

.. describe:: A (type: vector<double>,double, class: HHGateF2D)

   lookupA: Compute the A gate value from two doubles, passed in as a vector.
   This is same as `alpha(V)` the gate transition rate from closed to open
   state in the Hodgkin-Huxley formulation. Unlike HHGate2D, HHGateF2D uses
   formula evaluation to get more accurate value, which is also slower.

.. describe:: B (type: vector<double>,double, class: HHGateF2D)

   lookupB: Compute B gate value from two doubles in a vector. This is same
   as `alpha(V)+beta(V)` in the Hodgkin-Huxley formulation, where alpha(V)
   is the gate transition rate from closed to open and beta(V) is the transition
   rate from open to closed state.
