311constexpr const char* useClock =
312 R"(Assign a clock-tick to specified objects.
313
314The sequence of clockticks with the same dt is
315according to their number. This is utilized for controlling the order of
316updates in various objects where it matters.
317
318Parameters
319----------
320tick: unsigned int
321 Tick number to assign
322path: str
323 Path of target objects. Use a wildcard to select all
324 elements on a subtree, or for selecting by condition.
325fn: str
326 Function (dest field) to be execeuted on tick
327
328Examples
329--------
330In multi-compartmental neuron model a compartment's membrane potential (Vm)
331is dependent on its neighbours' membrane potential. Thus it must get the
332neighbour's present Vm before computing its own Vm in next time step. This
333ordering is achieved by scheduling the `init` function, which communicates
334membrane potential, on tick 0 and `process` function on tick 1.
335
336>>> moose.useClock(0, '/model/compartment_1', 'init')
337>>> moose.useClock(1, '/model/compartment_1', 'process'));
338)";
