15/**
16 * Base class for all object lookups.
17 * Provides the MOOSE interface so that it handles messaging, class info,
18 * and basic fields. Does not do data.

29 * name is its name
30 * numData is the number of data entries, defaults to a singleton.

32 * be replicated on all nodes, or partitioned without replication.

134 /**
135 * Looks up specified field data entry. On regular objects just
136 * returns the data entry specified by the rawIndex.
137 * On FieldElements like synapses, does a second lookup on the
138 * field index.
139 * Note that the index is NOT a
140 * DataIndex: it is instead the raw index of the data on the current
141 * node. Index is also NOT the character offset, but the index
142 * to the data array in whatever type the data may be.

144 * The DataIndex has to be filtered through the nodeMap to
145 * find a) if the entry is here, and b) what its raw index is.

414 /**

416 * Each index (BindIndex) gives a vector of MsgFuncBindings,
417 * which are just pairs of ObjId, FuncId.
418 * SrcFinfo keeps track of the BindIndex to look things up.
419 * Note that a single BindIndex may refer to multiple Msg/Func
420 * pairs. This means that a single MsgSrc may dispatch data
421 * through multiple msgs using a single 'send' call.
422 */
423 vector< vector < MsgFuncBinding > > msgBinding_;
424
425 /**
426 * Digested vector of message traversal sets. Each set has a
427 * Func and element to lead off, followed by a list of target
428 * indices and fields.
429 * The indexing is like this:
430 * msgDigest_[ numSrcMsgs * dataIndex + srcMsgIndex ][ func# ]
431 * So we look up a vector of MsgDigests, each with a unique func,
432 * based on both the dataIndex and the message number. This is
433 * designed
434 * so that if we expand the number of data entries we don't have
435 * to redo the ordering.
436 */
437 vector< vector < MsgDigest > > msgDigest_;
