21/**
22 * The HHChannel class sets up a Hodkin-Huxley type ion channel.
23 * The form used here is quite general and can handle up to 3
24 * gates, named X, Y and Z. The Z gate can be a function of
25 * concentration as well as voltage. The gates are normally computed
26 * using the form
27 *
28 * alpha(V)
29 * closed <------------> open
30 * beta(V)
31 *
32 * where the rates for the transition are alpha and beta, and both
33 * are functions of V.
34 * The state variables for each gate (X_, Y_, and Z_) are
35 * the fraction in the open state.
36 *
37 * Gates can also be computed instantaneously, giving the instantaneous
38 * ratio of alpha to beta rather than solving the above conversion
39 * process.
40 * The actual functions alpha and beta are provided by an auxiliary
41 * class, the HHGate. The idea is that all copies of a channel share the
42 * same gate, thus saving a great deal of space. It also makes it
43 * possible to cleanly change the parameters of all the channels of
44 * a give class, all at once. Should one want to mutate a subset
45 * of channels, they just need to set up separate gates.
46 *
47 * HHGates are implemented as a special category of FieldElement, so that
48 * they can be accessed as readonly pointers available to the HHChannel.
49 * The FieldElement containing the HHGate appears as a child Element of
50 * the HHChannel. The HHChannel Element can be an array; the associated
51 * HHGate is a singleton. So there has to be a local copy of the HHGate
52 * on each node.
53 */
54

111 /**
112 * processFunc handles the update and calculations every
113 * clock tick. It first sends the request for evaluation of
114 * the gate variables to the respective gate objects and
115 * recieves their response immediately through a return
116 * message. This is done so that many channel instances can
117 * share the same gate lookup tables, but do so cleanly.
118 * Such messages should never go to a remote node.
119 * Then the function does its own little calculations to
120 * send back to the parent compartment through regular
121 * messages.
122 */
