13/**
14 * Loads in a kkit.g model.
15 * It makes a separate compartment for each distinct volume
16 * it finds in the system. The largest volume gets set to /model/kinetics.
17 * The next volume is in /model/compartment_1.
18 * The next volume is in /model/compartment_2.
19 * And so on.
20 * Has two options for handling compartments and solvers.
21 * By default, it puts the entire reaction system on /kinetics and makes
22 * just one solver.
23 * If the moveOntoCompartments_ flag is set it will put each model entity
24 * onto the appropriate compartment. It will create separate solvers for
25 * each compartment.
26 *
27 */

131 * We have a slight problem because MOOSE has a more precise value

133 * the volume relationships are unknown. So we need to fix up conc
134 * units of all pools reacs post-facto.
135 * Here we assume that the conc units from Kkit are
136 * meant to be OK, so they override the #/cell (lower case k) units.
137 * So we convert all the Kfs and Kbs in the entire system after
138 * the model has been created, once we know the order of each reac.
