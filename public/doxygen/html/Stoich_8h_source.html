13/**
14 * Stoich is the class that handles the stoichiometry matrix for a
15 * reaction system, and also setting up the computations for reaction
16 * rates. It has to coordinate the operation of a number of model
17 * definition classes, most importantly: Pools, Reacs and Enz(yme)s.
18 * It also coordinates the setup of a large number of numerical solution
19 * engines, or solvers: Ksolve, Gsolve, Dsolve, and SteadyState.
20 * The setup process has to follow a tight order, most of which is
21 * internally manged by the Stoich.
22 * The Stoich itself does not do any 'process' functions. It just sets up
23 * data structures for the other objects that do the crunching.
24 *
25 * 1. Compartment is set up to a cell (neuroMesh) or volume (other meshes)
26 * 2. Compartment assigned to Stoich. Here it assigns unique vols.
27 * 3. Dsolve and Ksolve assigned to Stoich using setKsolve and setDsolve.
28 * 3.1 At this point the Stoich::useOneWay_ flag is set if it is a Gsolve.
29 * 4. Call Stoich::setWildcard. All the rest happens internally, done by Stoich:
30 * 4.1 assign compartment to Dsolve and Ksolve.
31 * 4.2 assign numPools and compts to Dsolve and Ksolve.
32 * 4.3 During Zombification, zeroth vector< RateTerm* > is built.
33 * 4.4 afterZombification, stoich builds rateTerm vector for all vols.
34 * 4.5 Stoich assigns itself to Dsolve and Ksolve.
35 * - Ksolve sets up volIndex on each VoxelPool
36 * - Dsolve gets vector of pools, extracts DiffConst and MotorConst
37 * 4.6 Dsolve assigned to Ksolve::dsolve_ field by the Stoich.
38 * 5. Reinit,
39 * 5.1 Dsolve builds matrix, provided dt has changed. It needs dt.
40 * 5.2 Ksolve builds solvers if not done already, assigning initDt
41 *
42 * As seen by the user, this reduces to just 4 stages:
43 * - Make the objects.
44 * - Assign compartment, Ksolve and Dsolve to stoich.
45 * - Set the stoich path.
46 * - Reinit.
47 */
