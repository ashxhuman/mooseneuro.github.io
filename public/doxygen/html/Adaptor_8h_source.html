13/**
14 * This is the adaptor class. It is used in interfacing different kinds
15 * of solver with each other, especially for electrical to chemical
16 * signeur models.
17 * The Adaptor class is the core of the API for interfacing between
18 * different solution engines. It is currently in use for interfacing
19 * between chemical and electrical simulations, but could be used for other
20 * cases such as mechanical models.
21 *
22 *
23 * The API for interfacing between solution engines rests on
24 * the following capabilities of MOOSE.
25 * 1. The object-oriented interface with classes mapped to biological
26 * and modeling concepts such as electrical and chemical compartments,
27 * ion channels and molecular pools.
28 * 2. The invisible mapping of Solvers (Objects implementing numerical
29 * engines) to the object-oriented interface. Solvers work behind the
30 * scenes to update the objects.
31 * 3. The messaging interface which allows any visible field to be
32 * accessed and updated from any other object.
33 * 4. The clock-based scheduler which drives operations of any subset of
34 * objects at any interval. For example, this permits the operations of
35 * field access and update to take place at quite different timescales
36 * from the numerical engines.
37 * 5. The implementation of Adaptor classes. These perform a linear
38 * transformation::
39 *
40 * (y = scale * (x + inputOffset) + outputOffset )
41 *
42 * where y is output and x is the input. The input is the average of
43 * any number of sources (through messages) and any number of timesteps.
44 * The output goes to any number of targets, again through messages.
45 *
46 * It is worth adding that messages can transport arbitrary data structures,
47 * so it would also be possible to devise a complicated opaque message
48 * directly between solvers. The implementation of Adaptors working on
49 * visible fields does this much more transparently and gives the user
50 * facile control over the scaling transformatoin.
51 *
52 * These adaptors are used especially in the rdesigneur framework of MOOSE,
53 * which enables multiscale reaction-diffusion and electrical signaling
54 * models.
55 * As an example of this API in operation, I consider two mappings:
56 * 1. Calcium mapped from electrical to chemical computations.
57 * 2. phosphorylation state of a channel mapped to the channel conductance.
58 *
59 * 1. Calcium mapping.
60 * Problem statement.
61 * Calcium is computed in the electrical solver as one or more pools that
62 * are fed by calcium currents, and is removed by an exponential
63 * decay process. This calcium pool is non-diffusive in the current
64 * electrical solver. It has to be mapped to chemical calcium pools at a
65 * different spatial discretization, which do diffuse.
66 * In terms of the list of capabilities described above, this is how the
67 * API works.
68 * 1. The electrical model is partitioned into a number of electrical
69 * compartments, some of which have the 'electrical' calcium pool
70 * as child object in a UNIX filesystem-like tree. Thus the
71 * 'electrical' calcium is represented as an object with
72 * concentration, location and so on.
73 * 2. The Solver computes the time-course of evolution of the calcium
74 * concentration. Whenever any function queries the 'concentration'
75 * field of the calcium object, the Solver provides this value.
76 * 3. Messaging couples the 'electrical' calcium pool concentration to
77 * the adaptor (see point 5). This can either be a 'push' operation,
78 * where the solver pushes out the calcium value at its internal
79 * update rate, or a 'pull' operation where the adaptor requests
80 * the calcium concentration.
81 * 4. The clock-based scheduler keeps the electrical and chemical solvers
82 * ticking away, but it also can drive the operations of the adaptor.
83 * Thus the rate of updates to and from the adaptor can be controlled.
84 * 5. The adaptor averages its inputs. Say the electrical solver is
85 * going at a timestep of 50 usec, and the chemical solver at 5000
86 * usec. The adaptor will take 100 samples of the electrical
87 * concentration, and average them to compute the 'input' to the
88 * linear scaling. Suppose that the electrical model has calcium units
89 * of micromolar, but has a zero baseline. The chemical model has
90 * units of millimolar and a baseline of 1e-4 millimolar. This gives:
91 * y = 0.001x + 1e-4
92 * At the end of this calculation, the adaptor will typically 'push'
93 * its output to the chemical solver. Here we have similar situation
94 * to item (1), where the chemical entities are calcium pools in
95 * space, each with their own calcium concentration.
96 * The messaging (3) determines another aspect of the mapping here:
97 * the fan in or fan out. In this case, a single electrical
98 * compartment may house 10 chemical compartments. Then the output
99 * message from the adaptor goes to update the calcium pool
100 * concentration on the appropriate 10 objects representing calcium
101 * in each of the compartments.
102 *
103 * In much the same manner, the phosphorylation state can regulate
104 * channel properties.
105 * 1. The chemical model contains spatially distributed chemical pools
106 * that represent the unphosphorylated state of the channel, which in
107 * this example is the conducting form. This concentration of this
108 * unphosphorylated state is affected by the various reaction-
109 * diffusion events handled by the chemical solver, below.
110 * 2. The chemical solver updates the concentrations
111 * of the pool objects as per reaction-diffusion calculations.
112 * 3. Messaging couples these concentration terms to the adaptor. In this
113 * case we have many chemical pool objects for every electrical
114 * compartment. There would be a single adaptor for each electrical
115 * compartment, and it would average all the input values for calcium
116 * concentration, one for each mesh point in the chemical calculation.
117 * As before, the access to these fields could be through a 'push'
118 * or a 'pull' operation.
119 * 4. The clock-based scheduler oeperates as above.
120 * 5. The adaptor averages the spatially distributed inputs from calcium,
121 * and now does a different linear transform. In this case it converts
122 * chemical concentration into the channel conductance. As before,
123 * the 'electrical' channel is an object (point 1) with a field for
124 * conductance, and this term is mapped into the internal data
125 * structures of the solver (point 2) invisibly to the user.
126 *
127 *
128 *
129 */
